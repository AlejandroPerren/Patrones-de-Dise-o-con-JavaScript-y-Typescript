Explicación del Patrón Strategy

El patrón Strategy permite definir diferentes comportamientos (estrategias) para realizar una misma tarea. En este caso, se aplica para calcular montos de venta bajo diferentes condiciones: ventas regulares, con descuento o en el extranjero.

Razones para usar este patrón

  • Flexibilidad: Permite cambiar el comportamiento en tiempo de ejecución.
  • Mantenimiento: Cada estrategia está encapsulada en su propia clase, simplificando el mantenimiento.
  • Extensibilidad: Es fácil añadir nuevas estrategias sin modificar el código existente.
  • Eliminación de condicionales: No necesitas usar múltiples bloques if o switch.

Funcionamiento del código

El contexto (Salecontext) utiliza diferentes estrategias para calcular el monto. Puedes cambiar la estrategia en tiempo de ejecución mediante el método setStrategy().
